# User Story - US_033

## Story ID
   * US_033

## Story Title
   * Medication Conflict Detection and Alerts

## Description
  * As a clinical staff member, I want the system to automatically detect conflicts between medications, allergies, and patient conditions, so that dangerous drug interactions or allergic reactions are prevented before prescribing.

## Acceptance Criteria
  1. **Given** a patient profile contains medications list from intake or documents, **When** I add a new medication in the Clinical Data Review screen (SCR-010) or when a new medication is extracted from uploaded documents, **Then** the system analyzes all medications using OpenAI API with medical safety prompt, detects conflicts between existing medications (drug-drug interactions), checks medications against patient's allergy list (drug-allergy conflicts), validates medications against patient's conditions (contraindications like "Ibuprofen + Chronic Kidney Disease"), achieves >99% conflict detection accuracy per AIR-S03, displays CRITICAL red banner alert for high-severity conflicts (e.g., "Warfarin + Aspirin = increased bleeding risk"), displays WARNING yellow notification for moderate conflicts, shows NO CONFLICT green indicator for safe combinations, includes conflict details (mechanism, severity level 1-5, clinical guidance), logs all conflict checks to audit log with timestamp and checked_by_staff_id, and blocks "Save" button for critical conflicts until staff acknowledges warning with "Override Reason" text field.

## Edge Cases
   * What happens when medication name is unrecognized? (System searches drug database by partial match, prompts staff for clarification)
   * How are dosage-dependent interactions handled? (System checks dosage thresholds, shows "Safe at current dose" or "Unsafe above X mg/day")
   * What if patient has no known allergies recorded? (System shows "No allergy data - verify with patient" warning, allows proceeding)

> **Wireframe Details (Medication Conflicts):**
> - **Medications section**: Table columns - Medication Name, Dosage, Frequency, Conflict Status (Safe/Warning/Critical), Actions (Edit/Remove/Details)
> - **Conflict banner**: Appears at top of screen when conflict detected, red background for critical, yellow for warning, includes "View Details" link
> - **Conflict details modal**: Opens on click, shows: Interacting Medications (2 or more), Interaction Type (Drug-Drug/Drug-Allergy/Drug-Condition), Severity (1-5 scale with icons), Mechanism of Interaction (text explanation), Clinical Guidance (recommended actions), "Override" button with required reason field
> - **Add medication form**: Medication name autocomplete, dosage input, frequency dropdown, real-time conflict check on blur/change
> - **Alert persistence**: Critical alerts remain until override acknowledged, yellow warnings dismissible but logged
