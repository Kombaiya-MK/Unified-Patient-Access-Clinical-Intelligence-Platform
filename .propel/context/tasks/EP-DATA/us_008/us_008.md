# User Story - US_008

## Story ID
   * US_008

## Story Title
   * Database Performance Indexes Creation

## Description
  * As a developer, I want optimized indexes on frequently queried columns, so that database queries execute within performance targets (<100ms for slot availability, <200ms for patient lookup).

## Acceptance Criteria
  1. **Given** core tables are created, **When** I execute index creation script, **Then** indexes are created on timeslots(staff_id, date), appointments(patient_id, status, appointment_date), users(email), audit_logs(user_id, created_at), and query performance improves to target latencies.

## Edge Cases
   * What happens when creating index on large existing table? (Use CREATE INDEX CONCURRENTLY to avoid table locking in production)
   * How are unused indexes identified? (Monitor pg_stat_user_indexes, remove indexes with low idx_scan counts)
