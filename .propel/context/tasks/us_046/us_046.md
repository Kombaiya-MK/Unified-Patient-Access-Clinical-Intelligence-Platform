# User Story - US_046

## Story ID
   * US_046

## Story Title
   * Real-Time Dashboard Notifications System

## Description
  * As a user (Patient, Staff, Admin), I want to receive real-time notifications on my dashboard for important events like appointment updates, medication conflicts, and system alerts, so that I stay informed of critical information without needing to refresh or check multiple pages.

## Acceptance Criteria
  1. **Given** I am logged into my role-specific dashboard, **When** a notification event occurs (appointment reminder 24h before, status update, medication conflict, waitlist slot available), **Then** a notification popup appears in the top-right corner with icon (info/warning/critical), title, message, timestamp, and dismiss button without page reload via WebSocket connection.

  2. **Given** I have unread notifications, **When** I view the dashboard header, **Then** I see a bell icon with a red badge showing the count of unread notifications (e.g., "3"), and clicking the bell opens a notification history panel sliding from the right.

  3. **Given** I click a notification popup, **When** the notification is actionable (e.g., "Appointment confirmed - Add to calendar"), **Then** clicking the notification takes me to the relevant page (appointment details) or triggers the action (calendar sync), and marks the notification as read, decrementing the badge count.

  4. **Given** I receive a critical notification (medication conflict, insurance pre-check failure), **When** the popup displays, **Then** it remains visible until I explicitly click "Acknowledge" button, uses red color scheme with alert icon, and displays in notification history with "Critical" label.

## Edge Cases
   * What happens when the user is offline and misses real-time notifications? (On reconnection, fetch missed notifications from last sync timestamp via REST API and display catch-up banner)
   * How many notification popups can stack simultaneously? (Max 3 visible, older ones auto-dismiss after 10s for info priority, queue additional notifications)
   * What if the notification history panel has 100+ notifications? (Paginate 20 per page, infinite scroll, "Mark All as Read" and "Clear Read Notifications" buttons)
   * How are notification preferences managed? (User settings page allows enabling/disabling notification types per category)

> **Wireframe Details (Notification Popup):**
> - **Popup component**: 
>   - Position: Fixed top-right (desktop), top-center (mobile), 320px width
>   - Icon: Info (blue circle-i), Warning (yellow triangle-!), Critical (red diamond-!)
>   - Title: Bold, 14px font, truncated after 40 chars
>   - Message: Regular, 12px font, max 2 lines, "Read more" link if truncated
>   - Timestamp: Gray, 10px font, relative time ("2 min ago")
>   - Dismiss button: X icon, top-right corner, hover effect
>   - Auto-dismiss: Info (10s), Warning (15s), Critical (manual only)
> - **Bell icon (Header)**:
>   - Position: Top navigation bar, right side
>   - Badge: Red circle with white number, positioned top-right of bell icon
>   - Hover effect: Tooltip "You have 3 unread notifications"
>   - Active state: Bell icon highlighted when panel open
> - **Notification panel**:
>   - Slide-in from right, 400px width (desktop), full-screen (mobile)
>   - Header: "Notifications" title, "Mark All Read" + "Settings" icons, close button
>   - List: Scrollable, grouped by date (Today, Yesterday, This Week)
>   - Item: Icon, title, message preview, timestamp, unread indicator (bolddot)
>   - Empty state: "No notifications" with bell icon illustration
> - **Notification types**:
>   - Appointment reminders (Info, blue)
>   - Appointment status updates (Info, blue)
>   - Medication conflicts (Critical, red)
>   - Insurance pre-check failures (Warning, yellow)
>   - Waitlist slot availability (Info, green)
>   - System alerts (Warning/Critical based on severity)
> - **Accessibility features**:
>   - ARIA live region: role="alert" for critical, role="status" for info
>   - Screen reader announcement: "New notification: [title]"
>   - Keyboard navigation: Tab to focus popup, Enter to dismiss, ESC to close panel
>   - Focus trap: When panel open, focus cycles within panel only
