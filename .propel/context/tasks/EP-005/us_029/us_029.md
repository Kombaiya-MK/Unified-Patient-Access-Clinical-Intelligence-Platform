# User Story - US_029

## Story ID
   * US_029

## Story Title
   * AI Document Data Extraction Service

## Description
  * As a clinical staff member, I want uploaded documents automatically processed by AI to extract structured medical data, so that I don't have to manually read and transcribe information from PDFs and images.

## Acceptance Criteria
  1. **Given** a document is uploaded with status="Uploaded", **When** the background extraction job (Bull/BullMQ queue) picks up the document for processing, **Then** the system calls OpenAI Vision API (for images) or text extraction API (for PDFs/DOCX), sends document content with structured extraction prompt requesting JSON output with fields (patient_name, date_of_birth, document_date, diagnosed_conditions, prescribed_medications with dosage/frequency, lab_test_results with test_name/value/unit/reference_range, allergies, provider_name, facility_name), parses JSON response with >95% extraction accuracy per AIR-Q01, stores extracted data in PatientProfiles table with document_id reference, updates document status to "Processed" with extraction_completed_at timestamp, and if extraction confidence <90% for any field, flags for manual review with status="Needs Review".

## Edge Cases
   * What happens when document is illegible or corrupted? (AI returns error, set status="Extraction Failed", notify staff to upload clearer copy)
   * How are handwritten notes handled? (Use OCR preprocessing, flag low confidence fields, allow staff to manually correct extracted data)
   * What if OpenAI API is down? (Circuit breaker triggers after 3 failures, queue document for retry with exponential backoff: 5 min, 15 min, 1 hour)
