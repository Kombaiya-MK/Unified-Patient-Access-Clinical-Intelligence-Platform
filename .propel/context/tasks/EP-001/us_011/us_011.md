# User Story - US_011

## Story ID
   * US_011

## Story Title
   * Immutable Audit Logging System

## Description
  * As a compliance officer, I want all user actions logged immutably to audit_logs table with 7-year retention, so that the system maintains HIPAA-compliant audit trails for regulatory requirements.

## Acceptance Criteria
  1. **Given** a user performs any action (login, CRUD operations, data access), **When** the action completes, **Then** the system creates an audit log entry with user_id, action_type, resource_type, resource_id, timestamp, IP address, user_agent, and PII-redacted details (replacing names/emails/SSNs with patient_id/user_id), stores it in audit_logs table with INSERT-only permissions (no UPDATE/DELETE allowed), and retains for 7 years.

## Edge Cases
   * What happens when audit log insert fails? (Fail the primary operation with 500 error, log to separate error table for investigation)
   * How is PII redaction enforced? (Middleware intercepts all audit log writes, applies redaction rules before INSERT)
   * What if audit_logs table grows to millions of records? (Implement table partitioning by year, archive old partitions to cold storage)
