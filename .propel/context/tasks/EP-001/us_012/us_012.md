# User Story - US_012

## Story ID
   * US_012

## Story Title
   * Login Page and Authentication Flow UI

## Description
  * As a user (Patient, Staff, Admin), I want an intuitive and accessible login page with email/password inputs, so that I can securely access my role-specific dashboard.

## Acceptance Criteria
  1. **Given** I navigate to the application root URL, **When** I am not authenticated, **Then** the system displays the login page (SCR-001) with email and password fields, "Remember Me" checkbox, "Forgot Password" link, and "Login" button.

  2. **Given** I am on the login page, **When** I enter valid credentials for my role (Patient/Staff/Admin) and click "Login", **Then** the system authenticates me and redirects me to my role-specific dashboard (SCR-002 for Patient, SCR-003 for Staff, SCR-004 for Admin).

  3. **Given** I enter invalid credentials, **When** I click "Login", **Then** the system displays an inline error message "Invalid email or password" below the form without page reload, and logs the failed attempt.

  4. **Given** I am on the login page, **When** I tab through the form elements using keyboard only, **Then** all interactive elements (email field, password field, checkbox, links, button) receive visible focus indicators (2px solid outline) in logical order.

## Edge Cases
   * What happens when the user submits the form with empty fields? (Display inline validation errors: "Email is required", "Password is required")
   * How does the login page handle slow network responses? (Show loading spinner on button, disable form, timeout after 30 seconds with retry option)
   * What if the backend is temporarily unavailable during login? (Display "Service temporarily unavailable. Please try again." with retry button)
   * How is password visibility toggled? (Show/hide password icon button with aria-label for screen readers)
