# User Story - US_010

## Story ID
   * US_010

## Story Title
   * RBAC Middleware Implementation

## Description
  * As a developer, I want role-based access control middleware that validates JWT roles, so that API endpoints enforce Patient/Staff/Admin permissions automatically.

## Acceptance Criteria
  1. **Given** a user makes an API request with JWT token, **When** the RBAC middleware validates the token, **Then** the system extracts the role claim, checks if the user's role is authorized for the endpoint (configured via decorator or route config), and either grants access or returns 403 Forbidden with "Insufficient permissions" message.

## Edge Cases
   * What happens when JWT token is valid but role claim is missing? (Return 403 with "Invalid token: missing role claim", log security event)
   * How are multi-role endpoints handled? (Accept array of allowed roles: @RequireRoles(['Staff', 'Admin']))
