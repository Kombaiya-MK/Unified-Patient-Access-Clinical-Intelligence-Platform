# User Story - US_003

## Story ID
   * US_003

## Story Title
   * PostgreSQL Database Setup with pgvector Extension

## Description
  * As a developer, I want PostgreSQL 15+ with pgvector extension configured, so that the application can store structured data and perform vector similarity searches for AI features.

## Acceptance Criteria
  1. **Given** I have PostgreSQL 15+ installed, **When** I run the database initialization script, **Then** the UPACI database is created with pgvector extension enabled and connection test returns success.

  2. **Given** the database is initialized, **When** I inspect the schema, **Then** I see all core tables (Users, Appointments, ClinicalDocuments, PatientProfiles, AuditLogs, Departments, TimeSlots, Waitlist, Notifications) created with proper foreign key constraints.

  3. **Given** the database connection is configured, **When** the backend starts, **Then** it successfully connects to PostgreSQL using connection pooling (pg library) with max 20 connections and logs connection status.

  4. **Given** pgvector is enabled, **When** I insert a test embedding vector, **Then** the system successfully stores and retrieves it using cosine similarity search (<-> operator).

## Edge Cases
   * What happens when pgvector extension fails to install? (Provide installation guide for Windows/Linux and alternate fallback without vector search)
   * How does the system handle database connection failures during startup? (Retry 3 times with exponential backoff, then exit with error)
   * What if migrations fail midway? (Rollback to last successful migration, log error details)
