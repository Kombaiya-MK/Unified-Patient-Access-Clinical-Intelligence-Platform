# User Story - US_002

## Story ID
   * US_002

## Story Title
   * Node.js Backend API Setup with Express

## Description
  * As a developer, I want a properly configured Node.js Express backend with middleware and routing structure, so that API endpoints can be implemented efficiently with proper error handling.

## Acceptance Criteria
  1. **Given** I have the project repository, **When** I navigate to the backend/ directory and run `npm install`, **Then** Express 4.x, middleware (CORS, helmet, morgan), and dependencies install successfully.

  2. **Given** the backend is configured, **When** I run `npm run dev`, **Then** the Express server starts on port 3001 with nodemon watching for file changes and logging requests to console.

  3. **Given** the API structure is setup, **When** I inspect the codebase, **Then** I see organized routes/ (authentication, appointments, patients), middleware/ (auth, errorHandler, validation), and config/ (database, redis, openai) directories.

  4. **Given** error handling middleware is configured, **When** an unhandled API error occurs, **Then** the system returns a JSON error response with status code, message, and stack trace (development only) without crashing the server.

## Edge Cases
   * What happens when required environment variables (DB_URL, REDIS_URL, JWT_SECRET) are missing? (Server fails to start with clear error listing missing variables)
   * How does the system handle uncaught promise rejections? (Global handler logs error and returns 500 status with generic message)
   * What if Express port 3001 is already in use? (Try alternative ports 3002-3005 or display error)
