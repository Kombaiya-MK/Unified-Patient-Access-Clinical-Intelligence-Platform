# User Story - US_013

## Story ID
   * US_013

## Story Title
   * Patient Appointment Booking Interface

## Description
  * As a patient, I want to book appointments through an intuitive interface with calendar view and available time slots, so that I can schedule visits with preferred providers at convenient times.

## Acceptance Criteria
  1. **Given** I am logged in as a patient, **When** I navigate to the appointment booking page (SCR-006), **Then** I see a calendar view with available dates highlighted, department dropdown, provider dropdown (filtered by department), and time slot grid showing available slots for selected date.

  2. **Given** I select a date and time slot, **When** I click "Book Appointment" and confirm, **Then** the system creates the appointment, sends confirmation email with PDF attachment, updates my dashboard (SCR-002) with the new appointment card, and triggers calendar sync (Google/Outlook).

  3. **Given** my preferred time slot is unavailable, **When** I select "Add to my waitlist" for that slot, **Then** the system adds me to the waitlist and displays confirmation message "You'll be notified if this slot becomes available".

  4. **Given** I complete a booking, **When** the system generates the PDF confirmation, **Then** the PDF includes appointment date/time, provider name, department, location, patient name, appointment ID, and QR code for reference.

## Edge Cases
   * What happens when selected slot becomes unavailable between selection and submission? (Display error: "This slot was just taken. Please select another slot.", refresh available slots)
   * How does the system handle booking attempts outside business hours? (All slots shown are within configured business hours only)
   * What if calendar sync fails (Google/Outlook API error)? (Complete booking anyway, show warning "Calendar sync failed - add manually", retry button available)
   * How are same-day appointments handled? (Allow if slot available and >2 hours from current time)

> **Wireframe Details (SCR-006):**
> - **Calendar component**: Month view with available dates in green, unavailable in gray, selected in blue
> - **Department dropdown**: Searchable select with all active departments
> - **Provider dropdown**: Dynamically filtered based on selected department
> - **Time slot grid**: 30-minute intervals, available slots shown as buttons, booked slots disabled/grayed
> - **Slot details panel**: Shows selected date/time, provider name, estimated duration, "Book" + "Add to Waitlist" buttons
> - **Loading states**: Skeleton screens during slot loading (<500ms), spinner for booking action
> - **Responsive design**: Mobile (stacked layout), tablet (2-column), desktop (3-column with calendar/slots/details)
> - **Accessibility**: ARIA labels for all interactive elements, keyboard navigation (Tab to slots, Enter to select)
