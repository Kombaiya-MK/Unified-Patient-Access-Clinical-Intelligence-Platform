# User Story - US_048

## Story ID
   * US_048

## Story Title
   * Windows Services/IIS Deployment Configuration

## Description
  * As a DevOps engineer, I want to deploy the Node.js backend as a Windows Service and configure IIS reverse proxy for the React frontend, so that the platform runs reliably on Windows Server infrastructure with automatic restarts.

## Acceptance Criteria
  1. **Given** a Windows Server 2019/2022 environment, **When** I deploy the application, **Then** the system configures Node.js backend as Windows Service using node-windows or PM2 Windows service wrapper with service name "UPACI-Backend", sets service to start automatically on server boot with restart on failure (3 attempts with 30s delay), runs service under dedicated service account (LOCAL_SERVICE or custom domain account) with minimum required permissions, logs service output to Windows Event Log (Application log, source "UPACI") and file logs in C:\Logs\UPACI\, configures IIS as reverse proxy for React frontend on port 443 (HTTPS) with URL Rewrite module installed, creates IIS Application Pool "UPACI-Frontend" with .NET CLR version "No Managed Code", sets Application Pool identity to ApplicationPoolIdentity, configures reverse proxy rules in web.config: Frontend static files served from C:\inetpub\wwwroot\upaci-frontend\, API requests (/api/*) proxied to backend Node.js service at http://localhost:3000, WebSocket requests (/socket.io/*) proxied with WebSocket protocol support, installs SSL certificate (Let's Encrypt or commercial cert) and binds to HTTPS port 443, configures HTTPS redirect (HTTP to HTTPS) and HSTS headers (max-age=31536000), sets up Windows Firewall rules allowing inbound traffic on ports 80, 443, configures backend to accept X-Forwarded-For, X-Forwarded-Proto headers from IIS for correct client IP logging, implements health check endpoints (/health for backend, /healthcheck.html for frontend) monitored by IIS Application Initialization module, enables compression (gzip/brotli) in IIS for static assets and API responses reducing bandwidth by ~70%, configures URL Rewrite for single-page app routing (all frontend routes redirect to index.html except API calls), sets up process monitoring with PM2 or Windows Service SCM showing process CPU/memory usage, and documents deployment runbook in .propel/docs/windows-deployment.md with step-by-step configuration screenshots.

## Edge Cases
   * What happens when Node.js service crashes? (Windows Service Manager automatically restarts after 30s, alerts admin after 3 failed restart attempts)
   * How are SSL certificate renewals handled? (Manual renewal via IIS Manager or automated with certes/win-acme for Let's Encrypt, scheduled task checks expiry monthly)
   * What if IIS reverse proxy fails? (Backend Node.js still runs, IIS serves 502 Bad Gateway, health checks detect failure and trigger alert)

> **Implementation Notes:**
> - **Windows Service Setup**:
>   - Install `node-windows` npm package: `npm install -g node-windows`
>   - Create service install script `install-service.js`:
>     ```javascript
>     const Service = require('node-windows').Service;
>     const svc = new Service({
>       name: 'UPACI-Backend',
>       description: 'UPACI Platform Backend API Service',
>       script: 'C:\\Apps\\upaci-backend\\server.js',
>       nodeOptions: ['--max_old_space_size=4096'],
>       env: [{ name: "NODE_ENV", value: "production" }]
>     });
>     svc.on('install', () => svc.start());
>     svc.install();
>     ```
>   - Run installation: `node install-service.js` (requires admin privileges)
>   - Verify service in Services.msc: Status=Running, Startup Type=Automatic, Recovery=Restart on failure
> - **IIS Configuration**:
>   - Install IIS features: Web Server, Application Development (.NET Extensibility, ASP.NET), URL Rewrite 2.0, Application Request Routing (ARR)
>   - Create site in IIS Manager: Site name="UPACI-Frontend", Physical path="C:\inetpub\wwwroot\upaci-frontend", Binding=HTTPS port 443
>   - Configure web.config with URL Rewrite rules:
>     ```xml
>     <configuration>
>       <system.webServer>
>         <rewrite>
>           <rules>
>             <rule name="API Proxy" stopProcessing="true">
>               <match url="^api/(.*)" />
>               <action type="Rewrite" url="http://localhost:3000/api/{R:1}" />
>             </rule>
>             <rule name="SPA Fallback">
>               <match url=".*" />
>               <conditions logicalGrouping="MatchAll">
>                 <add input="{REQUEST_FILENAME}" matchType="IsFile" negate="true" />
>               </conditions>
>               <action type="Rewrite" url="/" />
>             </rule>
>           </rules>
>         </rewrite>
>       </system.webServer>
>     </configuration>
>     ```
>   - Enable WebSocket in ARR: Server Farms → Proxy → Enable WebSocket support
>   - Install SSL cert: IIS Manager → Server Certificates → Import or Request, bind to site on port 443
> - **Firewall Rules**: `New-NetFirewallRule -DisplayName "UPACI HTTPS" -Direction Inbound -Protocol TCP -LocalPort 443 -Action Allow`
> - **Monitoring**: Task Scheduler task to check service status every 5 minutes, send alert email if service stopped
