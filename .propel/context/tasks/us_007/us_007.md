# User Story - US_007

## Story ID
   * US_007

## Story Title
   * Core Database Schema Implementation

## Description
  * As a developer, I want all core database tables created with proper constraints and relationships, so that application data can be stored reliably and queried efficiently.

## Acceptance Criteria
  1. **Given** PostgreSQL database is initialized, **When** I run the schema migration scripts, **Then** all tables (Users, Appointments, ClinicalDocuments, PatientProfiles, AuditLogs, Departments, TimeSlots, Waitlist, Notifications) are created with primary keys, foreign keys, indexes, and check constraints without errors.

## Edge Cases
   * What happens when migration fails midway? (Transaction rollback ensures database consistency, migration status tracked in schema_migrations table)
   * How are schema changes managed in production? (Use migration tool like Flyway or node-pg-migrate for versioned migrations)
