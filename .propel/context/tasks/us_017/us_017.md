# User Story - US_017

## Story ID
   * US_017

## Story Title
   * Calendar Sync (Google/Outlook API Integration)

## Description
  * As a patient, I want my appointments automatically synced to my Google Calendar or Microsoft Outlook calendar, so that I have all my commitments in one place without manual entry.

## Acceptance Criteria
  1. **Given** I book/reschedule/cancel an appointment and have authorized calendar access (OAuth2 PKCE flow during first booking), **When** the appointment operation completes, **Then** the system calls Google Calendar API or Microsoft Graph API (based on user's connected account stored in users table calendar_provider column) to create/update/delete the corresponding calendar event with title "[Clinic Name] - Appointment with [Provider]", description (appointment details, location), start/end time (appointment_datetime and estimated duration), location field, and reminder set for 24 hours before, logs the sync status (calendar_synced_at timestamp), and retries up to 2 times on API failure with exponential backoff (5s, 15s delays).

## Edge Cases
   * What happens when patient's OAuth token expires? (Detect 401 Unauthorized from calendar API, mark calendar_sync_enabled=false, display notification "Calendar sync disconnected. Please reconnect in settings.")
   * How is calendar provider initially selected? (During first booking, show modal "Sync to calendar?" with Google + Outlook buttons, clicking initiates OAuth flow)
   * What if calendar API rate limit is exceeded? (Queue calendar sync operations in Redis, process in batch every 5 minutes to stay under rate limits)
