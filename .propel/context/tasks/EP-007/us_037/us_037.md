# User Story - US_037

## Story Title
   * US_037

## Story Title
   * Insurance Pre-check Service Integration

## Description
  * As a clinic administrator or front desk staff, I want to automatically verify patient insurance eligibility before appointments, so that billing issues are identified early and patients are notified of coverage problems.

## Acceptance Criteria
  1. **Given** a patient has an upcoming appointment scheduled, **When** the pre-check service runs (24 hours before appointment time via scheduled job), **Then** the system calls external insurance verification API (e.g., Availity, Change Healthcare) with patient_id + insurance_plan + member_id + appointment_date, receives eligibility response (Active/Inactive/Requires_Auth/Copay_Amount), stores verification result in database (insurance_verifications table with timestamp, status, copay, deductible_remaining, notes), displays verification status in staff queue screen (SCR-009) as badge (Green "Verified" / Red "Issue" / Yellow "Auth Required"), displays detailed verification info in patient profile (SCR-011 Insurance Info panel) including copay amount, coverage effective dates, authorization requirements, generates notification for staff if status is "Inactive" or "Requires_Auth" with patient contact reminder, retries failed API calls up to 3 times with exponential backoff (1min, 5min, 15min), and logs all verification attempts to audit log with API response codes.

## Edge Cases
   * What happens when insurance API is down? (System shows "Verification Pending" status, queues retry, sends alert to admin after 3 failures)
   * How are missing insurance details handled? (System flags patient record as "Insurance Info Incomplete", notifies staff to collect during check-in)
   * What if patient has multiple insurance plans? (System verifies primary insurance only, shows "Secondary Insurance" indicator for manual review)

> - **SCR-011 Patient Profile - Insurance Info panel**: Card layout showing Insurance Plan (text), Member ID (text), Status (badge from queue), Copay Amount ($XX.XX), Deductible Remaining ($XXX.XX), Coverage Dates (MM/DD/YYYY - MM/DD/YYYY), Authorization Notes (text area if required), Last Verified (timestamp), "Re-verify Now" button (triggers immediate API call)
