# User Story - US_039

## Story ID
   * US_039

## Story Title
   * Admin Dashboard with System Metrics

## Description
  * As a clinic administrator, I want a real-time dashboard showing key operational metrics and system health indicators, so that I can monitor clinic performance and proactively address issues.

## Acceptance Criteria
  1. **Given** I have admin role permissions, **When** I access the Admin Dashboard (SCR-004), **Then** the system displays real-time metrics updated every 30 seconds via WebSocket: Current Queue Size (number of patients waiting), Average Wait Time (minutes calculated from arrival to in-progress status), Today's Appointments (scheduled count, checked-in count, completed count, no-shows count), displays system health metrics: API Response Times (average ms for last 100 requests with target <500ms), AI Service Status (OpenAI API calls success rate, failures/min), Database Connection Pool (active connections / max connections), Redis Cache Hit Rate (%), displays operational KPIs for selected date range (default: last 7 days): Total Appointments, No-Show Rate (%), Average Booking Lead Time (days), Insurance Verification Success Rate (%), Patient Satisfaction Score (if survey data available), displays charts using Chart.js: Line chart for daily appointment volume, Bar chart for no-shows by day of week, Pie chart for appointment types distribution, allows filtering by date range (Today, Last 7 Days, Last 30 Days, Custom Range), allows exporting metrics as CSV for reporting, displays alerts section for system issues (Redis down, AI API quota exceeded, Database slow queries >5s), and updates metrics with smooth animations (no jarring refreshes).

## Edge Cases
   * What happens when WebSocket connection drops? (System falls back to polling every 60 seconds, shows "Real-time sync paused" warning)
   * How are historical metrics stored? (Daily aggregates saved to metrics table, raw data retained for 90 days per TR-006)
   * What if no data for selected date range? (Dashboard shows "No appointments in this period" message with option to select different range)

> **Wireframe Details:**
> - **Top metrics cards**: 4 cards in row - Current Queue (large number + trend arrow), Avg Wait Time (minutes + target indicator), Today's Appointments (fraction like "45/60"), No-Show Rate (% + daily comparison). Each card has icon, value, label, and sparkline mini-chart.
> - **System Health panel**: Traffic light indicators (Green/Yellow/Red) for API Speed (current ms), AI Service (success rate %), Database (connection %), Cache (hit rate %). Click indicator opens details modal with last 24h trend chart.
> - **Main charts section**: 3 charts - Line chart (Daily Appointments - 7 day view with hover tooltips), Bar chart (No-Shows by Weekday - Mon-Sun), Pie chart (Appointment Types - Online/Walk-in/Staff-booked with percentages)
> - **Date range selector**: Dropdown (Today/Last 7 Days/Last 30 Days/Custom) + calendar picker for custom, "Export CSV" button next to date selector
> - **Alerts section**: Red banner at top if critical issues, yellow warning box for non-critical, each alert has timestamp, description, "Dismiss" or "View Details" button
> - **Auto-refresh indicator**: Small icon in header showing "Live" status with green dot, shows last updated timestamp
