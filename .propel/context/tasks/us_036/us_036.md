# User Story - US_036

## Story ID
   * US_036

## Story Title
   * Admin Department and Provider Management

## Description
  * As a clinic administrator, I want to manage departments, providers, and their schedules from a centralized admin interface, so that appointment availability accurately reflects current clinic operations and staff assignments.

## Acceptance Criteria
  1. **Given** I have admin role permissions, **When** I access the Admin Dashboard (SCR-014), **Then** the system displays Departments table (columns: Department Name, Active Providers, Total Appointments, Status Active/Inactive), allows me to add new department with name + description + operating hours (Mon-Sun 8AM-8PM), allows me to edit existing departments including deactivation (soft delete), displays Providers table (columns: Provider Name, Specialty, Department, Availability Hours, Status), allows me to add new provider with name + specialty + department assignment + availability template (weekly recurring hours), allows me to edit provider schedules with visual calendar editor showing blocked time + available slots, allows me to assign providers to multiple departments if needed, saves all changes to database with audit log (who changed what when), refreshes appointment booking interface (SCR-006) availability within 10 seconds of schedule changes, and validates no overlapping provider assignments or double-bookings.

## Edge Cases
   * What happens when deactivating a department with future appointments? (System warns admin, offers to reassign appointments or notify patients)
   * How are provider schedule conflicts handled? (System shows existing appointments in red, prevents overlapping blocked time)
   * What if provider is removed but has upcoming appointments? (System requires reassignment of all future appointments before deletion)
