# User Story - US_009

## Story ID
   * US_009

## Story Title
   * User Authentication with OAuth2/JWT

## Description
  * As a user (Patient, Staff, Admin), I want to securely authenticate using OAuth2 with JWT tokens, so that my identity is verified and my session is managed securely with role-based access.

## Acceptance Criteria
  1. **Given** I am a registered user, **When** I enter valid credentials and submit the login form, **Then** the system generates a JWT token with 15-minute expiry containing my user ID, role, and email, and stores it in Redis with session management.

  2. **Given** I have a valid JWT token, **When** I make API requests to protected endpoints, **Then** the authentication middleware validates my token and grants access based on my role (Patient/Staff/Admin).

  3. **Given** my JWT token expires after 15 minutes, **When** I attempt to access a protected resource, **Then** the system returns 401 Unauthorized and redirects me to the login page with session timeout message.

  4. **Given** I am authenticated, **When** I click logout, **Then** the system invalidates my JWT token in Redis, clears the client-side token, and redirects me to the login page.

## Edge Cases
   * What happens when invalid credentials are submitted? (Return 401 with "Invalid email or password" message, log failed attempt to audit log)
   * How does the system handle concurrent login attempts from same user? (Allow multiple sessions, track last login timestamp)
   * What if Redis is unavailable during token validation? (Fail gracefully, return 503 Service Unavailable, log error)
   * How are brute force attacks prevented? (Rate limit: max 5 failed attempts per 15 minutes per IP address)
