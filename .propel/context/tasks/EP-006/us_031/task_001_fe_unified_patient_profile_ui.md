## Task Overview
Implement the Clinical Data Review screen (SCR-010) displaying the unified patient profile generated by the backend API (task_002). The UI presents aggregated patient data across seven sections (Demographics, Chief Complaint, Medical History, Current Medications, Allergies, Lab Results, Previous Visits) with conflict highlighting for data mismatches. Conflicts are visually indicated with yellow backgrounds for non-critical discrepancies and red alert banners for critical mismatches (e.g., DOB differences). AI-extracted fields display confidence scores as color-coded badges (green >90%, yellow 80-90%, red <80%). Each section shows "Last Updated" timestamps indicating data freshness. The screen handles edge cases: empty state when no documents uploaded, outdated medication warnings (>1 year old), and critical data conflict alerts requiring staff resolution.

## Implementation Plan
1. **Create ClinicalDataReview page**: Route `/staff/patients/:patientId/profile`, fetch unified profile from `GET /api/patients/:patientId/profile` (task_002)
2. **Implement ProfileHeader component**: Display patient photo, name, MRN, DOB, age, last visit date, "Last Updated: [timestamp]" global indicator
3. **Create ProfileTabs component**: Tab navigation for 7 sections (Demographics, Medical History, Medications, Allergies, Lab Results, Visits, Documents)
4. **Implement Demographics section**: Display name, DOB, gender, address, contact info, insurance; highlight conflicts with ConflictBadge
5. **Build Medications section**: List current medications with drug name, dosage, frequency, "Last Updated" date; show "Outdated?" warning for medications >1 year old
6. **Create LabResults section**: Display lab tests with date, value, unit, normal range; use recharts LineChart for trend graphs (e.g., glucose over time)
7. **Add conflict highlighting**: ConflictBadge component shows yellow background + "View Sources" dropdown listing conflicting documents; ConflictAlert shows red banner for critical mismatches (DOB, allergies)
8. **Implement confidence badges**: ConfidenceBadge displays confidence percentage with color coding (green ≥90%, yellow 80-89%, red <80%) next to AI-extracted fields
9. **Handle no documents edge case**: Show empty state with intake form data only + notice "No documents uploaded" + "Upload Documents" CTA button
10. **Add data source indicators**: Small document icons next to each data point with tooltip showing source document name and extraction date

**Focus on how to implement**:
- Use React Router for `/staff/patients/:patientId/profile` route with patientId dynamic param
- Fetch profile data on component mount: `useEffect(() => { fetchProfile(patientId).then(setProfile) }, [patientId])`
- Conditional rendering for conflicts: `{dataPoint.hasConflict && <ConflictBadge sources={dataPoint.sources} />}`
- recharts example: `<LineChart data={labTrends}><XAxis dataKey="date"/><YAxis/><Line dataKey="value" stroke={primary-600}/></LineChart>`
- Outdated medications check: `const isOutdated = dayjs(med.lastUpdated).isBefore(dayjs().subtract(1, 'year'))`
- Critical alert conditions: `if (profile.conflicts.includes('dob_mismatch')) { showCriticalAlert() }`
- Use design tokens: `error-600` for critical alerts, `warning-600` for conflict highlighting, `success-600` for confidence badges >90%
- Empty state: `{profile.documents.length === 0 && <EmptyState message="No documents uploaded" action={<UploadButton />} />}`
