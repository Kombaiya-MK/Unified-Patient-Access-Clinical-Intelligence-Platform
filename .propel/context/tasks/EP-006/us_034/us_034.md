# User Story - US_034

## Story ID
   * US_034

## Story Title
   * Clinical Data Review Interface

## Description
  * As a clinical staff member, I want a unified patient profile view that aggregates data from multiple documents with conflict highlighting and medical coding suggestions, so that I can quickly review patient history and prepare for consultations.

## Acceptance Criteria
  1. **Given** I am logged in as staff, **When** I access a patient's clinical data review page (SCR-010), **Then** I see a unified profile displaying Demographics, Chief Complaint, Medical History, Current Medications, Allergies, Lab Results, and Previous Visits sections with data aggregated from all uploaded documents.

  2. **Given** the system detects conflicting data across documents (e.g., different allergy listings), **When** the profile loads, **Then** conflicts are highlighted in yellow with diff view showing source documents, and I can select the correct version or merge data with "Resolve Conflict" action.

  3. **Given** the unified profile is displayed, **When** I view the ICD-10 and CPT coding section, **Then** the system shows AI-generated medical codes with confidence scores (e.g., "J45.909 Unspecified asthma - 92% confidence") and I can edit, approve, or reject each code.

  4. **Given** I view current medications, **When** the system detects potential medication conflicts, **Then** a critical alert banner displays "⚠ MEDICATION CONFLICT DETECTED" in red with drug interaction details (e.g., "Warfarin + Aspirin: increased bleeding risk") and recommended actions.

## Edge Cases
   * What happens when document extraction is still processing? (Show "Processing..." status with estimated time remaining, allow partial review of completed sections)
   * How does the system handle documents in different formats (PDF, DOCX, scanned images)? (US-029 handles extraction, this story assumes extracted data exists in database)
   * What if AI confidence for medical coding is <90%? (Mark as "Needs Review" with yellow badge, require manual verification before approval)
   * How are historical versions of the profile tracked? (Every conflict resolution creates audit log entry, "View History" button shows timeline of changes)

> **Wireframe Details (SCR-010):**
> - **Header section**: Patient name, MRN, DOB, photo, last visit date, alert banner for critical issues (medication conflicts)
> - **Tab navigation**: Demographics, Medical History, Medications, Allergies, Lab Results, Visits, Documents, Coding
> - **Conflict highlighting**: Yellow background on conflicting fields, "View Sources" dropdown showing document origins, "Resolve" button opens modal with diff comparison
> - **Medication conflict alert**: Red banner at top with drug names, interaction severity (High/Medium/Low), clinical guidance, "Acknowledge" button (logs staff acknowledgment)
> - **Medical coding panel**: Table with columns: Code (ICD-10/CPT), Description, Confidence %, Source, Status (AI Generated/Approved/Rejected), Actions (Edit/Approve/Reject)
> - **AI confidence indicator**: Green (≥90%), Yellow (<90%, needs review), Gray (manually entered)
> - **Edit mode**: Inline editing for all fields, "Save Changes" button, auto-save draft every 30s
> - **Document reference links**: Each data field shows source document(s), click to open document viewer
> - **Accessibility**: Screen reader announces conflict count on page load, keyboard shortcuts (C for resolve conflict, M for medication details)
