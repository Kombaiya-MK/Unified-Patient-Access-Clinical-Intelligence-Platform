# User Story - US_032

## Story ID
   * US_032

## Story Title
   * Medical Coding (ICD-10/CPT) Automation

## Description
  * As a clinical staff member, I want AI-generated ICD-10 and CPT codes suggested automatically based on patient's diagnoses and procedures, so that billing and insurance claims are coded accurately without manual code lookups.

## Acceptance Criteria
  1. **Given** a patient profile contains diagnosed conditions from documents or intake, **When** I access the Medical Coding tab in clinical data review (SCR-010), **Then** the system analyzes all diagnoses using OpenAI API with medical coding prompt, suggests ICD-10 codes for each condition (e.g., "Hypertension, unspecified" → I10), displays Code + Description + Confidence Score (%), achieves >98% mapping accuracy per AIR-S02, allows me to edit/approve/reject each code with dropdown showing alternative codes, saves approved codes to Appointments table (icd10_codes JSON array), logs all coding decisions to audit log with approver_staff_id, and marks coding as "Approved" or "Pending Review" based oni confidence threshold (auto-approve if >95% confidence, require review if <95%).

## Edge Cases
   * What happens when condition description is vague? (AI returns multiple possible codes ranked by likelihood, staff selects most appropriate)
   * How are combination codes handled? (System suggests primary code + secondary codes if condition requires multiple codes)
   * What if no matching ICD-10 code exists? (AI suggests closest match with "Best match" indicator, staff can search ICD-10 database manually)
