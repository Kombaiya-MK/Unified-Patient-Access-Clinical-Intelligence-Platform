# User Story - US_018

## Story ID
   * US_018

## Story Title
   * PDF Appointment Confirmation Generation

## Description
  * As a patient, I want to receive a beautifully formatted PDF confirmation after booking an appointment, so that I have a professional document for my records and insurance purposes.

## Acceptance Criteria
  1. **Given** I successfully book or reschedule an appointment, **When** the system generates the confirmation PDF (using pdfkit or similar library), **Then** the PDF includes clinic letterhead/logo, patient name and MRN, appointment date and time in large bold font, provider name and credentials, department and location with address, appointment ID and QR code (encoding appointment URL for quick lookup), appointment duration and type (consultation/follow-up/procedure), preparation instructions (fasting requirements, documents to bring), insurance information section, clinic contact phone/email, cancellation policy, and compliance footer ("Protected Health Information - Handle Securely per HIPAA"), saves to cloud storage/local filesystem with filename `confirmation_[appointment_id]_[timestamp].pdf`, and returns secure download URL valid for 7 days.

## Edge Cases
   * What happens when PDF generation fails (library error, memory issue)? (Log error, retry once, if still failing send text-only email with appointment details and display "PDF generation failed, you'll receive details via email")
   * How is PDF storage managed long-term? (Delete PDFs older than 30 days from storage to save space, patient can regenerate from appointment history)
   * What if patient doesn't have insurance information? (Mark insurance section as "Not provided - please bring insurance card to appointment")
